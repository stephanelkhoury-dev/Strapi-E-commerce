## Performance Optimizations

### 1. Static Generation (SSG)

Most pages are statically generated at build time:

```typescript
// Products and categories use generateStaticParams
export async function generateStaticParams() {
  const products = await getProducts();
  return products.map((p) => ({ slug: p.slug }));
}
```

### 2. Incremental Static Regeneration (ISR)

Content pages revalidate on a schedule:

| Page | Revalidate Interval |
|------|-------------------|
| Homepage | 1 minute |
| Categories | 1 hour |
| About, Contact, FAQ | 1 hour |
| Legal pages | 1 hour |

### 3. On-Demand Revalidation

When content changes in Strapi, a webhook triggers cache purge:

```
Strapi Save → POST /api/revalidate → revalidateTag() → Fresh CDN
```

### 4. Image Optimization

- Next.js `<Image>` component (automatic WebP/AVIF)
- `sharp` for server-side processing
- Remote patterns for `localhost:1337` and `res.cloudinary.com`
- Responsive `sizes` attribute on all images
- Lazy loading by default

### 5. Code Splitting

- Next.js automatic route-based code splitting
- Dynamic imports for heavy components:
  ```typescript
  const HeavyComponent = dynamic(() => import('./Heavy'), { ssr: false });
  ```

### 6. Bundle Optimization

- Tree-shaking via Turbopack
- No barrel file re-exports that could hurt tree-shaking
- Specific icon imports from lucide-react

### 7. Caching Strategy

| Resource | Cache Strategy |
|----------|---------------|
| Static pages | CDN edge cache (ISR) |
| API responses | `next/cache` with tags |
| Images | Immutable + CDN |
| JS/CSS bundles | Content-hashed filenames |
| Cart state | localStorage |

## SEO Implementation

### 1. Metadata API

Every page exports metadata:

```typescript
export const metadata: Metadata = {
  title: "Page Title | ShopNow",
  description: "Up to 160 characters",
  openGraph: {
    title: "Page Title",
    description: "OG description",
    images: [{ url: "/og-image.jpg" }],
  },
};
```

### 2. Dynamic Metadata

Product and category pages generate metadata from Strapi:

```typescript
export async function generateMetadata({ params }): Promise<Metadata> {
  const product = await getProduct(params.slug);
  return {
    title: product.seo?.metaTitle || product.name,
    description: product.seo?.metaDescription || product.shortDescription,
  };
}
```

### 3. Sitemap

Auto-generated by `next-sitemap`:

```
/sitemap.xml          → Sitemap index
/sitemap-0.xml        → All public URLs
/robots.txt           → Auto-generated
```

**Excluded paths:** `/auth/*`, `/account/*`, `/checkout`, `/cart`, `/order/*`, `/api/*`, `/search`

**Priority map:**
| Page | Priority |
|------|----------|
| Homepage | 1.0 |
| Products | 0.9 |
| Categories | 0.8 |
| Content pages | 0.6 |

### 4. Structured Data

Product pages include JSON-LD:

```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "iPhone 15 Pro",
  "description": "...",
  "image": "...",
  "offers": {
    "@type": "Offer",
    "price": "999.99",
    "priceCurrency": "USD",
    "availability": "https://schema.org/InStock"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.5",
    "reviewCount": "12"
  }
}
```

### 5. Security Headers

## Accessibility

### Semantic HTML

- `<header>`, `<main>`, `<footer>`, `<nav>`, `<aside>`
- `<h1>` → `<h6>` hierarchy on every page
- `<button>` for actions, `<a>` for navigation
- `<form>` with proper `<label>` associations

### ARIA

- `role="navigation"`, `role="contentinfo"`, `role="region"`
- `aria-label` on icon-only buttons
- `aria-expanded` on dropdowns and accordions
- `aria-hidden="true"` on decorative elements
- `aria-live="polite"` on toast notifications

### Keyboard Navigation

- All interactive elements focusable
- Visible focus indicators
- Escape key closes modals/drawers
- Tab order follows visual layout

### Color Contrast

- All text meets WCAG AA contrast ratios
- Focus indicators have 3:1 contrast minimum
- Don't rely on color alone to convey information
